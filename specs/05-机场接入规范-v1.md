# Swell 机场接入规范 v1

**Swell Provider Integration Spec**

**适用对象:** 希望为 Swell 用户提供更好体验的服务商
**当前状态:** 草案 v1
**核心原则:** **渐进增强,零破坏。** 什么都不做也能用;做得越多,你的用户体验越好。

---

## 0. 一句话

**你现在的订阅在 Swell 里已经能用了。** 这份规范讲的是:如果你愿意多做一点,能解决哪些你和你用户都头疼的问题。

**所有扩展都在独立端点上,不会动你现有的订阅** —— 其他客户端(sing-box / Clash / 小火箭)完全不受影响。

---

## 1. 为什么需要这份规范

代理生态里有几件事**一直没有标准**,导致所有客户端只能靠猜:

| 信息 | 现状 | 后果 |
|---|---|---|
| **线路倍率** | 没有任何标准字段,只能写在线路名里 | 客户端靠正则猜 `[2.0倍消耗]` / `0.5x` / `x2` …写法五花八门,猜错就是用户流量算错 |
| **线路地区** | 只能靠线路名里的 emoji 国旗 | 命名不规范就识别不了;而且**名字可能和真实落地不符** |
| **真实落地** | 无法从配置得知 | 你的「🇦🇹 奥地利」节点入口在新加坡 → 客户端查 IP 得到"新加坡" |
| **公告 / 账户信息** | 只能塞成假节点 | 你的广告混在节点列表里,会**参与测速和自动选择**;用户可能被自动连到一条广告上 |
| **流量 / 到期** | `subscription-userinfo` 头是约定俗成,不是标准 | 不返回就完全拿不到 |

**这些不是客户端的锅,是生态缺标准。** Swell 想把这件事做对。

### 对你的直接好处

- ✅ **线路名可以变干净** —— 不用再往名字里塞 `[2.0倍消耗]`、广告、账户信息
- ✅ **公告有正经位置** —— 不用再伪装成节点,也不会被用户当成"坏节点"投诉
- ✅ **少客诉** —— "为什么扣这么多流量""哪条能看 Netflix""为什么连不上"这些能在 App 里自己回答
- ✅ **落地地区显示准确** —— 你的中转/家宽线路不会被误标成入口地区

## 2. 三级支持

| 级别 | 你要做什么 | 成本 | 效果 |
|---|---|---|---|
| **Level 0** | 什么都不做 | 0 | 能用。Swell 靠解析线路名,准确度看你的命名规范 |
| **Level 1** | 正确返回几个 HTTP 头 | ~10 分钟 | 流量/到期/订阅名/面板入口 全部正确显示 |
| **Level 2** | 提供一个 meta 端点 | ~半天 | 完整支持:倍率精确、落地地区准确、公告独立、解锁标记 |

**建议至少做到 Level 1。** 大多数面板(V2board / Xboard / SSPanel)本来就支持,只是要确认没关掉。

## 3. Level 1 — HTTP 响应头(强烈建议)

拉订阅时返回这几个头即可:

```http
subscription-userinfo: upload=455727941; download=6603013641; total=107374182400; expire=1788220800
profile-update-interval: 24
profile-web-page-url: https://user.yourpanel.com
content-disposition: attachment; filename="蓝鲸云"
```

| 头 | 说明 | 不返回的后果 |
|---|---|---|
| `subscription-userinfo` | `upload`/`download` 字节数;`total` 总配额字节;`expire` 到期 Unix 时间戳 | **Swell 隐藏整个流量与到期区块**(不会显示"未知") |
| `profile-update-interval` | 建议更新间隔(小时) | 默认 24 |
| `profile-web-page-url` | 用户面板地址 | 不显示「打开面板」按钮 |
| `content-disposition` | filename 作为订阅名 | 用户要手输订阅名 |

**约定:**

- `expire = 0` 或不返回 → 视为**永不过期**
- `total = 0` → 视为**无限流量**
- 剩余流量 = `total − upload − download`

> ⚠️ 请注意:Swell 只在**拉订阅时**能拿到这些数字,所以会在 UI 上标注"数据更新于 N 小时前"。这不是 bug,是诚实。

## 4. Level 2 — Meta 端点(完整支持)

### 4.1 为什么是独立端点,而不是给配置加字段

**因为 sing-box 会拒绝未知字段。**

官方 sing-box(`option/options.go` → `decoder.DisallowUnknownFields()`)在解析配置时会**因为任何未知字段直接报错**。

也就是说:**你往 sing-box config 的 outbound 里加自定义字段,所有官方 sing-box 客户端都会解析失败。**

所以 Swell 的扩展全部走**独立端点**:

- 你现有的订阅**一个字节都不用改**
- 其他客户端**完全不受影响**
- Swell 拿到 meta 就用增强模式,拿不到就自动回退到解析线路名

### 4.2 端点定义

```http
GET  {你的订阅URL的同源路径}/swell/v1/meta?token={与订阅相同的token}
Accept: application/json
```

**推荐做法:** 在你现有订阅 URL 的基础上加路径。例如订阅是

```
https://api.example.com/sub?token=abc123
```

meta 端点就是

```
https://api.example.com/sub/swell/v1/meta?token=abc123
```

Swell 会**自动尝试**这个地址。404 / 超时 → 静默回退到 Level 1/0,不报错、不打扰用户。

### 4.3 响应格式

```json
{
  "version": 1,
  "updated_at": 1789000000,

  "subscription": {
    "name": "蓝鲸云",
    "plan": "标准套餐 100G/月",
    "traffic": { "used": 41339715174, "total": 107374182400 },
    "expire": 1788220800,
    "reset_at": 1790000000,
    "web_page": "https://user.example.com",
    "support": "https://t.me/example_support"
  },

  "announcements": [
    {
      "level": "info",
      "text": "官网已更新,请访问 example.com 获取最新地址",
      "url": "https://example.com",
      "expire": 1790000000
    }
  ],

  "nodes": [
    {
      "tag": "🇦🇹 奥地利[AWS] [1.0倍消耗]",
      "multiplier": 1.0,
      "region": "AT",
      "entry_region": "SG",
      "kind": "relay",
      "residential": false,
      "streaming": ["netflix", "disney", "hbo"],
      "load": 0.42,
      "status": "ok"
    },
    {
      "tag": "🇺🇸 美国住宅[*纯净家宽*] [5.0倍消耗]",
      "multiplier": 5.0,
      "region": "US",
      "entry_region": "JP",
      "kind": "residential",
      "residential": true,
      "streaming": ["netflix", "disney", "hulu"],
      "status": "ok"
    }
  ]
}
```

### 4.4 字段说明

#### `subscription`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | string | 否 | 订阅显示名 |
| `plan` | string | 否 | 套餐名,显示在订阅详情 |
| `traffic.used` / `traffic.total` | int | 否 | 字节。给了就以此为准(优先于 HTTP 头) |
| `expire` | int | 否 | Unix 时间戳。`0`/省略 = 不过期 |
| `reset_at` | int | 否 | **流量重置时间** —— HTTP 头表达不了,这是 Level 2 的增值 |
| `web_page` | string | 否 | 用户面板 |
| `support` | string | 否 | 客服入口 → Swell 会做成「联系客服」按钮 |

#### `announcements`

**这是给你的广告和账户信息的正经位置。**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `level` | enum | 否 | `info` / `warning` / `critical`,默认 `info` |
| `text` | string | **是** | 正文 |
| `url` | string | 否 | 点击跳转 |
| `expire` | int | 否 | 过期后不再显示 |

> 🔴 **请不要再把公告伪装成节点。**
> 我们实测过的机场里,`🇭🇰 官网:xxx.com` 这类假节点会:
> - 被解析成香港节点,进入「香港」地区组
> - **参与测速**,而 `server: 127.0.0.1` 的那种会返回极低延迟
> - **被自动选择选成"最快节点"** → 用户连上一条广告,以为你的服务坏了
>
> Swell 会尽力过滤,但**过滤是靠猜的,一定会有漏网或误伤**。用 `announcements` 是双赢。

#### `nodes[]`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tag` | string | **是** | **必须与订阅里 outbound 的 tag 完全一致**(逐字节),这是唯一的关联键 |
| `multiplier` | float | 否 | 流量倍率。`1.0` = 正常,`0.5` = 半价,`5.0` = 五倍 |
| **`region`** | string | 否 | **落地地区**,ISO 3166-1 alpha-2(`US` / `HK` / `JP`)。**这是本规范最重要的字段** |
| `entry_region` | string | 否 | 入口地区。用于向用户解释"香港入口 → 美国落地" |
| `kind` | enum | 否 | `direct` 直连 / `relay` 中转 / `residential` 家宽 / `iepl` 专线 / `info` 信息条 |
| `residential` | bool | 否 | 是否住宅 IP |
| `streaming` | string[] | 否 | **你已验证可解锁的服务**:`netflix` `disney` `hbo` `hulu` `prime` `youtube_premium` `chatgpt` `claude` |
| `load` | float | 否 | 当前负载 0–1 |
| `status` | enum | 否 | `ok` / `maintenance` / `down` |

**除 `tag` 外全部可选。** 给多少用多少。

### 4.5 🔴 `region` 为什么最重要

**因为客户端没有任何办法自己知道你的线路落地在哪。**

我们实测的一家机场:

```
🇦🇹 奥地利[AWS]   → server: awssg.gfw2500.com
🇬🇧 英国原生      → server: awssg.gfw2500.com
🇧🇷 巴西原生      → server: awssg.gfw2500.com
🇻🇳 越南原生      → server: awssg.gfw2500.com
```

**四个国家,同一个新加坡入口。** 客户端拿 `awssg.gfw2500.com` 去查 geoip,四个全返回"新加坡"。

所以客户端只有两条路:

1. **猜名字** —— 你改个命名格式就失效
2. **实测落地** —— 每条线路跑一次 IP 查询,慢、耗流量、还得依赖第三方

**你给一个 `region` 字段,这个问题就没了。** 而且你比任何人都更清楚你的线路落地在哪。

### 4.6 `streaming` 的价值

**内核只能测延迟,完全无法判断一条线路能不能解锁 Netflix。**

这意味着客户端要么不承诺(用户体验差),要么自建一整套探测系统(慢、易失效)。

**而你是知道的。** 给了这个字段,Swell 就能:

- 在「Netflix」的服务例外里**只列出你标了 `netflix` 的线路**
- 用户不用再一条条试

> **诚信提醒:** 请只标你**真正验证过**的。标了打不开 = 用户投诉你,不是投诉 Swell。

## 5. 分流规则(可选,默认不启用)

如果你有自己优化过的分流规则,可以提供:

```http
GET {订阅URL}/swell/v1/rules?token={token}
```

```json
{
  "version": 1,
  "name": "蓝鲸云推荐分流",
  "description": "针对本机场线路优化的分流规则",
  "updated_at": 1789000000,
  "rule_sets": [
    {
      "tag": "streaming",
      "format": "binary",
      "url": "https://cdn.example.com/rules/streaming.srs"
    }
  ],
  "rules": [
    { "rule_set": ["streaming"], "outbound_hint": "region:US" }
  ]
}
```

### ⚠️ Swell 的策略:**默认不启用,而且不可覆盖用户设置**

| 规则 | 说明 |
|---|---|
| **默认关闭** | 用户必须在订阅详情页**明确开启**「使用本机场推荐分流」 |
| **优先级最低** | 永远排在 `场景规则 > 用户自定义 > 拦截 > 服务例外 > 地区/兜底` **之后** |
| **不能覆盖用户** | 用户设的服务例外永远优先 |
| **`outbound_hint` 是建议不是命令** | 用 `region:US` 这种抽象表达,由 Swell 翻译成实际出站。**你不能直接指定 outbound tag** |
| **不接受 reject** | 你不能通过规则屏蔽任何东西 |

**为什么这么保守:** 用户装 Swell 是为了用 Swell 的分流。机场规则劫持用户体验是这个生态的老问题(Karing 专门做了个「禁用[机场]分流规则」开关就是证据)。我们宁可默认关。

## 6. 线路命名建议(Level 0 也适用)

即使你不做 Level 2,**规范命名也能显著提升识别率**:

### ✅ 推荐

```
🇭🇰 香港 IEPL-01
🇯🇵 日本 东京-02 [2x]
🇺🇸 美国 洛杉矶-01 [0.5x]
```

- **emoji 国旗放最前** —— 这是最可靠的地区标识
- **倍率用 `[Nx]`**,统一格式
- 地区名 + 城市/线路编号

### ❌ 避免

```
Premium-x9                       ← 无任何地区标识 → 归入"未识别地区"
🇭🇰 官网:xxx.com                 ← 假节点,请用 announcements
剩余流量:990.38 GB               ← 同上
🇺🇸 美国-高速03                   ← 若实际落地香港,会误导用户(请给 region 字段)
```

> 💡 **做了 Level 2 之后,你的线路名可以彻底变干净** —— 倍率、公告、账户信息全都有了专门的字段,名字里只留地区和编号即可。

## 7. 缓存与频率

| 项 | 约定 |
|---|---|
| meta 端点调用频率 | 与订阅更新同步,**不会更频繁** |
| 缓存 | Swell 会缓存 meta,遵守你的 `Cache-Control` |
| 失败处理 | 404 / 5xx / 超时 → **静默回退**,不打扰用户 |
| 超时 | 10 秒 |
| `version` 不匹配 | 忽略未知版本,回退 |

## 9. FAQ

**Q: 我什么都不做,Swell 能用吗?**
能。靠解析线路名,准确度取决于你的命名规范。

**Q: 加了这些字段会不会影响我在小火箭/Clash 的用户?**
不会。扩展全在独立端点,你的订阅内容完全不变。

**Q: 为什么不直接在 sing-box config 里加字段?**
sing-box 会**拒绝未知字段**(`decoder.DisallowUnknownFields()`)。加了会导致所有官方 sing-box 客户端解析失败。

**Q: `tag` 一定要完全一致吗?**
是。逐字节一致,包括 emoji 和空格。这是关联节点的唯一方式。

**Q: 我可以只做一部分吗?**
可以。除 `tag` 外全部可选,给多少用多少。

**Q: `region` 和 `entry_region` 有什么区别?**
`region` 是**落地**(目标网站看到的 IP 归属),`entry_region` 是**入口**(用户连接的服务器)。中转和 CDN 线路两者不同。**用户关心的是 `region`。**

**Q: 我不想暴露入口地区。**
`entry_region` 可以省略。只给 `region` 就够。
