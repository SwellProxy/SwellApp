# Swell 订阅解析规格(给开发)

**版本:** v1
**基于:** 两份真实机场订阅的实测(wjkc66 / liangxin)+ sing-box v1.13.14 源码核实

> 这份是**防御性解析规格** —— 应付现实中机场订阅的烂摊子。
> 另有一份《Swell 机场接入规范》是给机场看的主动规范,两者配合:机场按规范做 → 走快路;不做 → 回退到本文的解析逻辑。

---

## 1. 支持的订阅格式

实测两家机场,格式完全不同,**都必须支持**:

| 格式 | 样例来源 | Content-Type | 特征 |
|---|---|---|---|
| **A. 完整 sing-box config JSON** | wjkc66 | `text/plain` | 以 `{` 开头,含 `outbounds` / `dns` / `route` |
| **B. base64 分享链接列表** | liangxin | `text/html` | base64 解码后每行一条 `vless://` / `hysteria2://` / `ss://` / `trojan://` |

### 检测顺序

```
1. 取回 body,trim
2. 若以 "{" 开头 → 尝试 JSON.parse
   2a. 含 "outbounds" 字段 → 格式 A(sing-box config)
   2b. 含 "proxies" 字段   → Clash YAML/JSON(可选支持)
3. 否则尝试 base64 解码
   3a. 解码后含 "://" → 格式 B(分享链接列表)
4. 否则:直接按分享链接列表解析(有些机场不 base64)
5. 都失败 → 报「订阅格式不支持」
```

> ⚠️ **不要依赖 Content-Type**。liangxin 返回的是 `text/html`,但内容是 base64。

## 2. 🔴 格式 A 的关键决策:只取 outbounds,其余全部丢弃

wjkc66 返回的不是节点列表,是**一整份 sing-box 配置**,它替你定了:

```json
"dns":   { "servers": [...], "fakeip": {...}, "final": "alidoh" }
"inbounds": [ { "type": "tun", "mtu": 9000, "stack": "system" } ]
"route": { "rules": [ {"protocol":"quic","action":"reject"}, ... ] }
"experimental": { "cache_file": {...} }
```

**注意 `{"protocol":"quic","action":"reject"}`** —— 它把全局 QUIC 掐了(为了强制 YouTube 走 TCP)。

### 决策:**只提取 `outbounds` 里 type 不是 selector/urltest/direct/block 的那些(即真实节点),其余整份丢弃。**

理由:

1. **机场会绑架用户设置**。它的 route 规则会覆盖 Swell 的「智能分流」和「服务例外」——用户在 Swell 里配的东西直接失效
2. **它的 TUN 参数是给它自己的客户端调的**(`mtu: 9000`、`stack: system`),不一定适合 Swell
3. **机场改配置 = Swell 行为突变**,而且我们无法测试
4. 用户装 Swell 是为了用 Swell 的分流,不是机场的

DNS / route / inbounds / experimental **全部用 Swell 自己的**。

> 对应 Karing 的做法:它有个「禁用[机场]分流规则」开关。我们更进一步——**默认就不用,连开关都不给**(v1)。

## 3. 订阅元数据:HTTP 响应头

```
subscription-userinfo: upload=455727941; download=6603013641; total=107374182400; expire=1788220800
profile-update-interval: 24
profile-web-page-url: https://user.example.com
content-disposition: attachment; filename="蓝鲸云"
```

| 字段 | 用途 | 缺失时 |
|---|---|---|
| `upload` + `download` | 已用流量 | 见 §5.4 fallback |
| `total` | 总配额 | 同上 |
| `expire` | 到期 Unix 时间戳 | **0 或缺失 = 不过期**,隐藏到期显示 |
| `profile-update-interval` | 机场建议的更新间隔(小时) | 默认 24 |
| `profile-web-page-url` | 用户面板 → 做成「打开面板」按钮 | 隐藏该按钮 |
| `content-disposition` filename | **订阅名自动填充** | 让用户手输 |

**剩余 = `total` − `upload` − `download`**

> ⚠️ 这是**约定俗成,不是标准**。必须能优雅处理:头缺失、`total=0`、`expire=0`、字段顺序乱、有无空格。
> ⚠️ **只在拉订阅那一刻拿得到。** 数据新鲜度 = 上次拉订阅的时间 → 时间戳必须和数字绑定显示。

## 4. 🔴 节点名解析(地区 / 倍率)

**内核既没有"地区"字段,也没有"倍率"字段**(已核实 sing-box v1.13.14 源码)。两者**只能从节点名解析**。

### 4.1 地区:靠 emoji 国旗

两家机场都用 emoji 国旗,这是事实标准:

```
🇭🇰 香港[电信] [1.0倍消耗]
🇯🇵日本高速01|CTCU|0.5x
```

**解析规则(按优先级):**

1. **Regional Indicator 组合**(U+1F1E6–U+1F1FF 两个连续码点)→ 直接映射 ISO 3166-1 alpha-2
2. 中文国名/地区名关键词:`香港|台湾|日本|新加坡|美国|韩国|英国|德国|法国|…`
3. 英文/缩写:`HK|TW|JP|SG|US|KR|UK|GB|DE|FR|…`(注意词边界,别把 `US` 匹配到 `AUSTRIA`)
4. 城市名:`洛杉矶|东京|首尔|伦敦|法兰克福|…`
5. 全失败 → **标记为「未识别地区」**,不要猜

> **「未识别地区」是必然会发生的**,UI 必须有兜底入口(屏 14 已有)。

### 4.2 倍率:两种写法都要吃

实测两家写法完全不同:

```
机场 A:🇺🇸 美国住宅[*纯净家宽*] [5.0倍消耗]
机场 B:🇬🇧英国伦敦|CTCU|0.1x
```

**正则(合并):**

```regex
[\[\(（|]?\s*(?:x|X|×)?\s*(\d+(?:\.\d+)?)\s*(?:x|X|×|倍消耗|倍率|倍)\s*[\]\)）|]?
```

需要覆盖:`[2.0倍消耗]` `[5.0倍消耗]` `0.1x` `0.5x` `2x` `x2` `×2` `(1.5倍)` `2.0倍率`

**规则:**

- 解析不到 → **不显示任何倍率**(不要写 1x)。"没写"和"就是 1x"分不清
- 解析到 `1.0` / `1` → **也不显示**(绝大多数是 1x,显示了是噪音)
- 解析到 `≠1` → **显示,用警告色**。包括 **`0.1x` / `0.5x` 这类小于 1 的**——这是好事,用户会想用便宜的

### 4.3 ⛔ 绝对不要用 geoip 查 server 地址来判断地区

**这是系统性错误,不是精度问题。** 实测证据:

**机场 A** —— 八个不同国家的节点,同一个新加坡入口:

```
🇦🇹 奥地利[AWS]     → awssg.gfw2500.com
🇬🇧 英国原生        → awssg.gfw2500.com
🇹🇷 土耳其原生      → awssg.gfw2500.com
🇹🇭 泰国原生        → awssg.gfw2500.com
🇰🇭 柬埔寨原生      → awssg.gfw2500.com
🇧🇷 巴西原生        → awssg.gfw2500.com
🇻🇳 越南原生        → awssg.gfw2500.com
🇰🇿 哈萨克斯坦      → awssg.gfw2500.com
```

**机场 B** —— 28 条节点(香港/日本/新加坡/美国/英国/韩国/台湾)全挂在 `cfyes.lxy1015.top` 和 `aws-linkN.liangxin1.xyz` 上。

geoip 查 server 会把这些**全部**归成新加坡 / Cloudflare。

**用户关心的是出口地区**(Netflix 和 Claude 看到的落地 IP),不是入口。
→ **地区只能靠名字解析,或实测落地检测。**

## 5. 🔴 信息条过滤(机场往节点列表塞广告)

### 5.1 实测发现

两家机场都在节点列表里塞**假节点**:

```
机场 A:🇭🇰 官网：任意.快车.com       ← 广告,但是可连的真实 vless 配置
机场 A:🇭🇰 永久流量:917.2G          ← 账户信息
机场 B:剩余流量：990.38 GB           ← 账户信息,server = 127.0.0.1
机场 B:套餐到期：长期有效             ← 账户信息,server = 127.0.0.1
```

### 5.2 不处理会怎样

- 前两条带 🇭🇰 → 被解析成**香港节点**,进「香港」地区组
- 参与 **urltest 测速**(127.0.0.1 那条会秒回 → **成为"最快"节点**)
- 参与**自动选择** → **用户可能被自动连到一条广告上**
- 「香港」地区节点数多算 2 条

> ⚠️ `127.0.0.1` 那条在 urltest 里会立刻返回极低延迟,**一定会被选成最快**。这是必须修的 bug,不是优化。

### 5.3 过滤规则

按顺序判定,命中任一即标记为 **`kind = info`**:

1. **server 是 `127.0.0.1` / `localhost` / `0.0.0.0` / `::1`** → 必定是信息条
2. **节点名匹配以下关键词**(不区分大小写):
   ```
   官网|官方|订阅|续费|购买|充值|套餐|到期|过期|剩余|流量|重置
   群组|频道|客服|工单|通知|公告|活动|优惠|邀请|注册|发布页
   http://|https://|www\.|\.com|\.net|\.xyz|t\.me|@
   ```
3. 名字里**不含任何地区标识**且**含数字+单位**(如 `917.2G` `990.38 GB`)

**`kind = info` 的节点:**

- ❌ 不参与测速
- ❌ 不参与自动选择 / urltest 组
- ❌ 不计入地区分组的节点数
- ❌ 不可被用户选中
- ✅ **可以在订阅详情页当「机场公告」展示** —— 这反而是个贴心功能

> 别直接删掉。用户想看"剩余流量 990.38 GB"这种信息,只是不该出现在节点列表里。

### 5.4 意外收获:信息条可以当元数据 fallback

机场 B **可能不返回 `subscription-userinfo` 头**,但它把配额塞在了节点名里:

```
剩余流量：990.38 GB
套餐到期：长期有效
```

**当 `subscription-userinfo` 缺失时**,可以从 info 类节点名里正则提取流量和到期作为 fallback:

```regex
剩余流量[：:]\s*([\d.]+)\s*(GB|G|TB|T|MB|M)
套餐到期[：:]\s*(.+)          # "长期有效" → 不过期
```

拿到了就正常显示;拿不到 → 订阅卡走**精简态**(整块隐藏进度条与到期,见屏 7)。

## 6. 协议支持核对

两家机场用到的,sing-box v1.13.14 全部原生支持:

| 协议 | 变体 | 实测出现 |
|---|---|---|
| VLESS | ws + tls + utls 指纹 | A、B |
| VLESS | Reality + `xtls-rprx-vision` | A、B |
| VLESS | tcp + Reality | B |
| Hysteria2 | + salamander obfs | A |
| Hysteria2 | + `pinSHA256` + `mport`(端口跳跃) | B |

**注意点:**

- **`mport=60000-65530`** —— 机场 B 用了**端口跳跃**。sing-box 的 hysteria2 outbound 用 `server_ports` 字段表达,分享链接里的 `mport` 要转换过去
- **`insecure: true`** —— 机场 A 有几条 hysteria2 开了跳过证书校验。原样透传,但可以在节点详情页给个小提示
- **`packet_encoding: xudp`** —— A 全都带,原样透传
- **utls 指纹** `safari` / `chrome` / `ios` —— 原样透传

## 7. 边界情况清单

| 情况 | 处理 |
|---|---|
| 订阅返回 HTML 错误页 | 检测不到 `://` 且不是 JSON → 报「订阅链接无效或已过期」 |
| base64 padding 缺失 | 补 `=` 到 4 的倍数再解 |
| 行尾是 `\r\n` | 两家都是,trim 每行 |
| 节点 tag 重复 | 去重时**保留第一条**,后续加后缀 `-2` |
| 节点名超长 | 列表中截断,详情页全显示 |
| `expire = 0` 或缺失 | **不过期**,隐藏到期显示 |
| `total = 0` | 无限流量,隐藏进度条 |
| 订阅拉取失败 | 保留上次数据 + 灰化 + 「N 天前的数据 · 点击重试」 |
| outbounds 为空 / 全是 info 条 | 报「该订阅没有可用线路」 |

## 8. 解析产物(内部数据结构建议)

```
Subscription {
  id, name, url
  traffic: { used, total } | null      // null = 拿不到,UI 整块隐藏
  expire: timestamp | null             // null = 不过期
  updated_at: timestamp                // 必填,UI 要显示"N 小时前"
  stale: bool                          // updated_at 超过阈值
  web_page_url: string | null
  announcements: [ { text, url } ]     // 来自 info 类节点
  nodes: [ Node ]
}

Node {
  tag                                  // 原始 outbound tag,唯一键
  display_name                         // 去掉倍率标记后的名字(可选)
  protocol                             // vless / hysteria2 / ...
  region: ISO-3166-1-alpha-2 | null    // null = 未识别地区
  region_source: "name" | "verified"   // 名字推测 vs 实测落地
  multiplier: float | null             // null = 未解析到,不显示
  kind: "node" | "info"                // info 不参与测速与选择
  raw_outbound: object                 // 原样传给内核
}
```

## 9. 关键结论速查

1. **两种格式都要支持**,别信 Content-Type
2. **格式 A 只取 outbounds**,其余整份丢弃 —— 否则机场绑架用户设置
3. **地区和倍率只能解析名字** —— 内核没有这两个字段
4. **绝不用 geoip 查 server 地址** —— 实测证明是系统性错误
5. **必须过滤 info 条** —— 否则 `127.0.0.1` 那条会被选成最快节点
6. **info 条可当元数据 fallback**,也可当公告展示
7. **倍率两种写法都要吃**,`0.1x` 这种小于 1 的要显示
8. **subscription-userinfo 是约定不是标准**,必须能降级
