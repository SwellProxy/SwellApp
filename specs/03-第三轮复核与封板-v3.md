## 2. 🔴 P0-1:3a 区没跟 v3 同步(旧设计残留)

### 问题

屏 9 升到 v3、屏 10 升到 v2 之后,**3a 区里引用它们的地方全是旧的**:

| 屏 | 现状(旧) | 应该是(v3) |
|---|---|---|
| **屏 15** | 区块标题「**服务分流**」+ 副文案「给常用服务指定专属线路」+ 「**全部 26 项 ›**」 | 「**服务例外**」+「想让某个服务特殊处理?」+ **无「全部 N 项」** |
| **屏 15** | 置灰列表是 Claude / YouTube / Netflix(旧预设) | Claude / ChatGPT / Gemini / X / TikTok(5 条新预设,全 🔒) |
| **屏 16** | 「添加服务或网站」+「服务 / 网站」tab 的旧流程 | 应与**屏 10-v2 推荐库**同一个流程 |

**这是上一轮「屏 9 与屏 9-v2 并存」的同一个错误。** 开发看到屏 15 会以为「服务分流 / 全部 26 项」还存在。

### 要求

不只是改这两屏 —— **全画布搜一遍**,凡是出现下列字样的地方全部替换:

- 「服务分流」(作为屏 9 的区块名时)→「服务例外」
- 「全部 N 项」→ 删除
- 「给常用服务指定专属线路」→「想让某个服务特殊处理?」
- 旧的预设组合(YouTube / Netflix / Telegram 出现在**预设**里)→ 换成 5 条新预设

> 注:屏 13「服务分流」作为**二级页标题**是可以保留的,那是页面名不是区块名。要改的是屏 9 / 屏 15 里的**区块标题**。

## 3. 🔴 P0-2:国际化(新决策)

**Swell 确定要做国际化。** 这不只是加一个「国家与地区」选项 —— 它会污染现有的好几处设计。

### 3.1 「流量去向」图解的例子是硬编码中国的

屏 9-v3 现在写:

```
国内网站与服务   微信、淘宝、银行            →  🏠 直连
国外网站与服务   YouTube、Instagram、Chat…  →  🚀 走线路
```

**对伊朗、俄罗斯用户,「微信、淘宝、银行」这三个例子毫无意义。**

要求:设计上标注这三个例子是**跟随所选国家动态替换**的变量,并至少给出 2 个国家的示例稿(中国 / 伊朗),让开发知道这块是可变的。

### 3.2 「国内」这个词要改

**「国内」假设了"用户人在那个国家"。** 但存在这些情况:

- 用户人在美国,选了「中国」(回国线路场景)→「国内直连」是错的
- 用户人在伊朗 →「国内」= 伊朗,不是中国

**要求:用具体国名,不要用「国内 / 国外」。**

- ❌ 国内网站与服务 / 国外网站与服务
- ✅ **{国家}网站与服务** / **其他国家的网站与服务**
  (中国版显示「中国网站与服务」,伊朗版显示「伊朗网站与服务」)

同样要检查:屏 4「国外走线路 · 国内直连」、屏 5/12 网络体检「国内直连 ✓ / 国外访问 ✓」、屏 9 三个模式的所有副文案。

### 3.3 「国家与地区」放哪

**它不是高级设置。** 它决定"什么算本国 → 直连",**选错整个智能分流全废**。把一个"错了就全毁"的开关塞进「普通用户无需进入」的页面,逻辑不通。

要求两处都要有:

1. **首次引导里问一次** —— 在屏 1 和屏 2 之间加一屏(或并入屏 1)。因为规则集要跟着国家下载,必须在导入订阅前就确定
2. **设置 › 通用** 里可改 —— 屏 11 的「通用」组加一项「国家与地区 · 中国 ›」

**不要放进「高级设置(自定义规则)」。**

### 3.4 界面语言 ≠ 分流国家

这是两个**独立**设置,不要绑在一起:

- **界面语言**:App 显示什么语言(设置 › 外观 或 通用)
- **国家与地区**:什么算本国 → 直连(设置 › 通用)

用户可能人在中国但用英文界面;也可能界面中文但要伊朗分流。

## 4. 🟡 P1 问题

### P1-1:屏 4 未连接态未优化(上轮草图未落地)

**现状问题:未连接和已连接是两个信息架构不同的页面,不是同一页面的两个状态。**

点一下按钮,「当前线路」从底部小卡跳进中间大卡,「分流模式」也跟着跳 —— **整页重排**。同一个信息在两态下位置不同,用户每次都要重新找。而且未连接态中间一大片空白,已连接态却塞满。

**另外「当前线路」这个词有歧义** —— 还没连,哪来的"当前"?

**优化方案:卡片位置固定、内容生长。而那片空白用来做这个 App 最有说服力的一件事。**

未连接态那一格显示「**你现在的 IP**」`🇨🇳 中国 · 223.104.x.x`(警示色),连上后**同一格**变成「**出口 IP**」`🇭🇰 香港 · 103.152.36.18`(品牌青)。

这一下同时解决三件事:空白有了内容、用户有了点按钮的**理由**、连上后有了**确认**。

详见提示词 4。

### P1-2:优先级声明不统一

现在有两处在声明优先级,拼不起来:

- 屏 17:「场景规则命中时**优先于服务分流**」
- 屏 16:「优先级定死:**自定义 > 拦截 > 服务 > 地区/兜底**」

**需要一条完整的、唯一的优先级链**,并且在两处引用同一个说法。建议:

```
场景规则 > 用户自定义 > 拦截 > 服务例外 > 地区/兜底
```

请在屏 9-v3 的「高级设置」页或「场景规则」页给出一次完整声明,其他地方引用它。

### P1-3:屏 6「管理 ›」入口确认

原来分组头右侧是「管理 ›」,现在变成「剩 61.5 GB · 47 天 ›」。**管理入口是整行可点吗?** 如果是,没问题,但请在标注里写明;如果不是,管理入口就丢了。

### P1-4:屏 10-v2 的已添加态

AI 组现在列的是 Claude[已添加] / Grok / Perplexity。但 **ChatGPT 和 Gemini 也是预设的**,它们应该也在 AI 组里并显示「已添加 ✓」。TikTok 同理(在视频组)。请确认是滚动截断还是真的漏了。

## 6. 可交互原型

**能做。** 当前设计本身就是一个 HTML 文件(`Swell UI.dc.html`),而且已经有 Tweaks 参数面板,说明具备状态切换能力。做成可点击原型只是加路由和状态。

**价值**:开发能点着看真实流程,而不是对着静态图猜交互——这对"封板后完全按 UI 开发"是刚需。

### 提示词 5 — 做成可交互原型

```

目标:开发和测试可以点着走完主要流程,不用对着静态图猜交互。

【必须可交互的路径】
1. Tab 栏:连接 / 节点 / 分流 / 设置 —— 四个 Tab 可互相切换
2. 连接页:
   - 点大按钮 → 未连接 ⇄ 已连接(含连接中的过渡态)
   - 点「换线路 ›」→ 跳到节点页

- 单个 HTML 文件,不依赖外部服务
- 所有数据用假数据,写死在文件里即可
- 状态切换要有过渡动画(与 iOS 观感一致)

  「国家 = 中国 / 伊朗」(用于验证国际化文案与举例的替换)
- 顶部保留一个屏号索引,可直接跳到任意屏
```

## 7. 封板前:状态补全清单

**当前 17 屏画的几乎全是"一切正常"的状态。** 若照此封板,开发一撞到异常就只能自己编,"完全按 UI 开发"会变成空话。

下面每一项都是**必然会发生**的,请在封板前补齐(可以合并到原型里):

| 位置 | 缺失状态 | 什么时候触发 |
|---|---|---|
| **屏 2** | 「跳过」之后的空 App | 按钮就摆在那儿,用户一定会点。没订阅时节点页长什么样?点大按钮会怎样? |
| **屏 2** | 订阅链接无效 / 导入失败 / 导入中 | 粘错、链接过期、格式不支持、网络断 |
| **屏 3** | 用户点「**不允许**」 | 100% 会发生。iOS 上一旦拒绝 VPN 配置,App 基本废了,必须引导去系统设置 |
| **屏 4/5** | 连接中 / 连接失败 / 断线重连中 | 节点挂了、订阅过期、网络不通 |
| **屏 4** | 流量**用完**(不是剩 8%)、订阅**已过期** | 必然发生 |
| **屏 6** | 无任何订阅(空状态)、全部线路超时、线路名超长截断 | |
| **屏 7** | 订阅拉取失败、订阅名超长 | |
| **屏 9** | 服务例外被用户全部删光(空状态) | |
| **屏 10** | 推荐库搜索无结果 | |
| **屏 14** | 选了某地区但**该地区一条线路都没有** | 用户只有香港节点却选了韩国 |
| **屏 17** | Wi-Fi 直连清单为空 | 首次进入 |

**封板标准**:上表每一项都有稿 → 逐项打勾 → 打满才叫封板。

## 9. 附录:内核红线(未变)

| 想要的 | 现实 |
|---|---|
| **iOS 按 App 开关代理** | ⛔ 做不到。只能按域名/IP。屏 10 删掉浏览器与国内 App 是对的,**别加回来** |
| 自动挑"能解锁 Netflix"的线路 | ⛔ 内核只测延迟。目前不承诺"解锁" |
| 又要自动选最快、又要 IP 不变 | ⛔ 互斥。🔒 / 🔄 就是在表达这个 |
| 内核知道线路在哪个国家 | ⛔ 不知道。靠解析线路名猜,所以「未识别地区」兜底必须留 |
| 内核有"倍率"字段 | ⛔ 没有。靠解析线路名,所以只显示 ≠1x 的 |
| 流量 / 到期 | ✅ 来自订阅 HTTP 响应头,**只在拉订阅那一刻拿得到** → 时间戳必须挂着 |
| 按 Wi-Fi / 蜂窝 / 低数据模式分流 | ✅ iOS 支持,屏 17 已用上 |
| **未连接时测速** | ⚠️ v1 决定:**测速会自动连接**(与 Karing 一致)。所以未连接态的延迟必须标「· 2 分钟前测」 |
| 链式代理 | ⚠️ v1 **不做**。v1.1 再说 |
